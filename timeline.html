Featured project

HarveStar Lunar Harvest End Effector

A Canadian Space Agency client project with Queen’s University to design, build, and test a lunar greenhouse harvesting end effector. The selected crab-claw design used a fixed arm, a servo-driven moving arm, elastic-band gripping surfaces, and an emergency-stop enabled path-planning program. I completed the project code and SolidWorks simulations; the CAD model itself was modeled by a teammate.

End-effector design

The claw used a simple asymmetric gripping strategy: one fixed side, one moving side, and compliant bands to handle different plant geometries with fewer moving parts and easier maintenance.

Control & validation

CircuitPython control refined pickup paths, release motions, speed, grip angles, and an emergency-stop routine while SolidWorks simulation checked load response and likely high-strain areas.

Engineering workflow

  1. Defined the lunar greenhouse harvesting context, including reduced gravity, tight workspace, safety, and plant-handling constraints.
  2. Evaluated gripper concepts and selected a crab-claw end effector for simplicity, grip adaptability, and maintainability.
  3. Ran SolidWorks stress/displacement simulation under gripping load to assess whether the design stayed structurally reasonable.
  4. Programmed path planning, servo movements, release motions, and emergency stop behavior, then iterated through physical testing.

Featured project

Water Filtration Prototype

A working water-treatment prototype combining mechanical design, CAD modeling, 3D-printed parts, Arduino/C control, pump-driven flow, two-step alum dosing, magnetic mixing, staged filtration, and clean-water collection. I wrote the project code independently.

Design logic

The SolidWorks assembly helped organize the pump, base plate, dosing tower, magnetic mixing system, filtration path, and receiving container before physical assembly and testing.

Control & build

Arduino/C logic coordinated the liquid-level check, alum dispenser, DC-motor mixing, and pump-driven transfer; code adjustments corrected the dosing sequence to deliver the target alum amount.

Process flow

  1. Pump untreated water into the process structure.
  2. Use the dispenser to introduce alum for coagulation and sedimentation.
  3. Drive a gear-based magnetic system at the bottom so the internal magnetic mixing bar can sync with the changing magnet position.
  4. Pump water onward through multi-layer filtration and collect cleaner output in a final container.

Featured project

Femoral Head Indexing Station

A manufacturing-cell redesign project for moving a prosthetic femoral head through laser etching and surface mapping positions. The system used recycled materials, motor characterization, encoder feedback, Arduino/C control, Excel data collection, and Python plotting. I wrote all project code independently.

Mechanical system

A belt-and-lever transmission converted motor rotation into controlled angular indexing while keeping the femoral head restrained only at the lower contact band.

Controls & testing

Encoder feedback, duty-cycle testing, and control logic were used to verify position, reduce slip effects, execute dwell times, and support emergency-stop behavior.

Engineering workflow

  1. Defined the required 108° clockwise, 198° counterclockwise, and 90° clockwise indexing sequence.
  2. Collected motor speed, torque, and duty-cycle data in Excel, then used Python to plot and interpret performance.
  3. Programmed the microcontroller in Arduino/C to control motor movement, dwell timing, encoder-based position checks, and emergency stopping.
  4. Iterated the recycled-material transmission until the mechanism could move predictably and verify position within a tight angular tolerance.
